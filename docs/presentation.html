Leadership Architect
Problem

Every producer reinvents Iceberg. Multi-region makes it worse.

Six stacks, six sets of quirks

  • Glue / EMR PySpark, ECS / Lambda Python, ECS / Lambda Java
  • Each re-learns the same Iceberg configuration traps
  • No shared code, no shared review checklist

Multi-region under strict rules

  • No cross-region S3. No MRAPs.
  • Metadata repointing is non-trivial and error-prone
  • Reference material assumes the opposite posture

No platform leverage

  • Compaction / snapshot expiry / orphan cleanup diverge per team
  • Security review reopens from zero on every onboarding
  • Policy cannot be enforced in one place
RESULT 10–12 person-weeks per producer — repeated across the mesh.

What Iceshelf is

A Claude Code plugin that writes Iceberg code into your repo.

9 skills

Slash commands that drive each stage of onboarding — from profile capture through multi-region DR.

5 subagents

Specialized reasoning: architect, code-generator, validator, multi-region planner, orchestrator.

Presets + profiles

Org-wide policy as YAML. Each producer inherits it; only overrides go in the profile.

YOU GET BACK

PySpark / PyIceberg / Java pipeline code  ·  Terraform for S3 / Glue / IAM / KMS / VPC  ·  CloudWatch alarms + dashboards  ·  DR runbook  —  all committed to your repo, reviewable in a normal PR.

Claude Code runs locally. Files land in your repo. Your CI/CD deploys.

│  ├─ pipeline/*.py          # Iceshelf: ingest + upsert jobs
│  └─ maintenance/*.py       # Iceshelf: compaction, expiry
├─ infra/iceberg/*.tf         # Iceshelf: S3 / Glue / IAM / KMS
└─ runbooks/failover.md       # Iceshelf: DR runbook

How it flows

1
Install the Iceshelf plugin in your Claude Code CLI.
Nothing leaves your machine. No account to sign up for.
2
Run /iceberg-profile-setup in the producer repo.
Captures region, bucket, stack, DR posture → profiles/<producer>.yaml.
3
Run /iceberg-onboard — orchestrator drives the rest.
Generates DDL, pipeline code, IaC, runbooks into the tree on the left.
4
Review the PR. You own every line.
Your existing CI/CD deploys exactly as it does for any other code change.
5
Ongoing: slash commands for day-2 operations.
Schema evolution, multi-region, maintenance — same pattern, same repo.
Iceshelf never runs at your runtime · Iceshelf never sees your data · Iceshelf has no cloud account of its own.

Architect Leadership
Security & compliance

Defensible by default — built for security review, not around it.

Identity

  • IAM scoped to DB + table ARNs
  • Never "*" in generated policies
  • Read / write / maint roles split
  • Lake Formation supported

Encryption

  • S3 = aws:kms by default
  • CMK alias pinned in profile
  • Per-producer key isolation
  • Key-grant list scripted

Network

  • VPC endpoints: S3 / Glue / STS
  • KMS + CloudWatch private
  • No public egress required
  • Private-subnet-only supported

Audit

  • Profile + code diff-reviewable
  • Required tags enforced
  • Optional WORM audit bucket
  • Retention windows explicit
FRAMEWORK ALIGNMENT
SOC 2 · HIPAA / PHI · PCI-DSS · FedRAMP / GovCloud  —  aligned by design; artifacts portable to GovCloud with a profile swap.

Leadership
The ask · 90-day pilot

Two producers. One quarter. Clear decision gate at week 12.

Scope

  • Two producers — one PySpark, one PyIceberg
  • One region pair (e.g. us-east-1 / us-west-2)
  • One preset (aws-glue-datamesh-strict by default)
  • Dry-run first — PR-reviewed in sandbox before prod

Timeline

Wk 1–2Preset sign-off; sandbox; profiles
Wk 3–7Producer #1: onboard → ingest → multi-region
Wk 8–9Producer #2 on same preset (reuse proof)
Wk 10Game-day: simulated primary-region outage
Wk 11–12Security / cost review → go/no-go gate

Exit criteria (all must pass)

  • Both producers in production, zero hand-edits
  • ≤ 2 person-weeks / producer (vs 10–12 DIY)
  • RPO ≤ 20 min and RTO ≤ 60 min on game-day
  • Security + cost review signed off

What we need from you

  • Executive sponsor (clears org-level blockers)
  • Platform-team lead: 0.5 FTE × 12 weeks
  • Two producer teams: ~0.25 FTE each
  • Sandbox AWS account with target region pair
